#7 Decoding · COLD 기출 P5 재출제 대비
  • AR decoding: sequential(병렬 불가), EOS/max-len까지, softmax 분포에서 샘플
  • Temperature: softmax 이전 로짓 rescale; τ↓=deterministic, τ→0=greedy
  • Top-k=고정 k개 / Top-p=누적확률 동적 후보(분포 모양 따라 변함)
  • Speculative: draft는 여전히 AR, 큰 모델은 병렬 verify, 출력 분포 보존
  • Test-time compute: 추론 전략(파라미터 고정), fine-tuning 아님
  • COLD(Qin, NeurIPS 2022): 제약→에너지 함수, 이산 \(\boldsymbol{y}\)→연속 로짓 \(\tilde{\boldsymbol{y}}\) 완화, Langevin으로 업데이트, training-free
  • 알고리즘 4단계: 초기화→에너지 정식화→N회 Langevin→discretization
  • Lexical 제약 \(E_1\): fluency 항들이 키워드 막삽입(reward hacking) 방지, ~95% coverage
\( E(\boldsymbol{y})=-\sum_i \lambda_i f_i(\boldsymbol{y}) \), \( p\propto e^{-E} \) · \( \tilde{\boldsymbol{y}}^{(n+1)}=\tilde{\boldsymbol{y}}^{(n)}-\eta\nabla E+\epsilon^{(n)} \)
"Decoding can be parallelized across all tokens at inference." → False (sequential)
"Temperature is applied after the softmax." → False (before, on logits)
"Top-p is a special case of beam search with p=1." → False (sampling ≠ search)
프롬프팅=implicit·무보장 ↔ COLD=explicit 에너지 최적화 / 제약 만족↑ = 에너지↓ = 확률↑

  • RLHF 한계 3: implicit goodness / OOD·adversarial query / 모델↑일수록 레이블 곤란
  • H vs H: helpfulness·harmlessness는 competing objectives → jointly maximize; PMP=community votes 사전학습
  • Constitutional AI(Anthropic 22.12): 사람은 헌법만 작성; SL-CAI=critique→revision→SFT / RL-CAI=principle-conditioned RM→RL(=RLAIF)
  • Self-Rewarding LM(ICML 2024): generator 겸 judge로 반복 자기학습
수식 없음 — instance-wise/pairwise, SL/RL 단계, bias↔해법 매칭이 전부

G-Eval=instance-wise 점수 ↔ Judge=pairwise 비교 / SL-CAI=SFT 데이터 생성 ↔ RL-CAI=AI 선호 레이블(RLAIF)
#9 RAG 최대 분량 챕터
  • 동기: 정적 데이터셋 → knowledge cutoff(GPT-4o 2023-10, LLaMA3 2023-12)·hallucination; 새 지식 fine-tuning은 기존 능력 손상
  • RAG = retrieve-and-read에서 generator만 LLM 교체; inference-based vs training-based
  • PageRank: \(1-d\)=random jump 확률; 의미 정보 안 씀 / BM25: sparse, 무학습, IDF는 corpus 전체 기준
  • DPR: 인코더 2개+dot product+contrastive loss(labeled \(p^+\)+in-batch negatives), OOD 약함 / Contriever: 같은 문서 crop=positive(자기지도), loss는 DPR과 동일, OOD 강건
  • REPLUG: LM frozen(black-box), 문서별 독립 prepend→확률 가중 앙상블; LSR=retrieval↔LM likelihood KL 최소화로 검색기만 학습
  • IRCoT: CoT 1스텝↔검색 교차; 마지막 스텝만 질의, 구절은 누적; multi-hop QA
  • RetRobust: NLI 필터(passage=premise, Q+A=hypothesis; not-entailed→direct answering) + 소량 학습
  • Self-RAG(ICLR 2024 Oral): SFT로 4 reflection 토큰(Retrieve/IsREL/IsSUP/IsUSE); GPT-4→critic(90% agree)→generator; 검색 청크 loss 제외
  • Search-R1(25.03): RL, outcome reward만; <think>/<search>/<information>/<answer>; retrieved token masking; interleaved 검색 자동 창발
  • 지표: EM=완전일치 0/1 / F1=단어 수준 / Recall@K=검색기 지표
\( \text{sim}(q,p)=E_Q(q)^{\top}E_P(p) \) · REPLUG: \( p(y|x,\mathcal{D}')=\sum_d p(y|d\circ x)\lambda(d,x) \)
"Dense retrievers always outperform BM25." → False (OOD에선 BM25가 이기기도)
"LSR maximizes the KL divergence." → False (minimize — 두 분포 일치)
"IRCoT uses the full accumulated CoT as query." → False (last step only)
BM25=lexical·무학습 / DPR=supervised dense / Contriever=self-supervised dense · Self-RAG=SFT ↔ Search-R1=RL
#10 Tool Use Toolformer vs ToolLLM
  • 동기: 정확한 계산·실시간 정보·외부 행동은 LLM 단독 불가; retrieval(RAG) ⊂ tool use; agent 3요소=tool use/memory/planning&action
  • Function calling(prompting): 모델은 이름+인자만 반환, 실행은 개발자 측; 이름 일관성+상세 설명 둘 다 중요; large LLM에서만 신뢰
  • Toolformer(NeurIPS 2023 Oral): self-supervised — "useful tool = next-token loss를 낮추는 호출"; in-context로 후보 샘플→실행→loss filtering→표준 LM loss로 FT
  • GPT-J 6B+5 tools가 GPT-3 능가(이득은 tool call 자체에서); 비용이 도구 수에 비례→확장 불가
  • ToolLLM(ICLR 2024): RapidAPI 3,451 tools / 16,464 APIs(tool 1개=API 여러 개); N개 subsample로 instruction 생성; DFSDT로 solution path 주석, "Finish" 도달분만 golden
  • 추론=RAG 유사 2단계(API retrieval→generation); 자체 BERT dense retriever > Ada·BM25; Pass=완수율 / Win=ChatGPT 대비 품질; retrieved top-5 > ground-truth(diversity)
filtering: \( L_i^- - L_i^+ \ge \tau \Rightarrow \) keep, \( L_i^-=\min\big(L_i(\varepsilon),\,L_i(e(c_i,\varepsilon))\big) \)
"Toolformer keeps a call when \(L_i^+ - L_i^- \ge \tau\)." → False (부호 반대)
"Toolformer checks whether the API answer is factually correct." → False (loss 감소만 봄)
"ToolLLM filters data by loss reduction like Toolformer." → False (ChatGPT가 실제로 풀어 Finish 도달분만)
Toolformer=loss로 거르는 self-supervised 5-tool ↔ ToolLLM=ChatGPT가 풀어 거르는 16K-API SFT+DFSDT
#11 LLM Agents 기말 핵심·3강 분량
  • Agent = LLM(brain) + Planning&Action + Tools + Memory; "not simply improved by scaling-up"
  • LLaVA: vision encoder→\(\mathbf{Z}_v\)→projector \(\mathbf{W}\)→\(\mathbf{H}_v\); Stage1=projector만 / Stage2=projector+LLM, encoder는 항상 frozen; 데이터는 GPT-4 생성
  • ReAct(ICLR 2023): reasoning+acting interleave, few-shot prompting(학습 아님); CoT-SC→ReAct는 득표<n/2일 때 backoff
  • Reflexion(NeurIPS 2023): verbal RL — Actor→Evaluator(ALFWorld에선 heuristic>GPT)→Self-reflection→Memory, weight update 없음
  • Plan-and-Solve: zero-shot "plan 먼저" 프롬프트 / ToT(NeurIPS 2023 Oral): 분해→생성→평가(value/vote, LLM 프롬프팅)→탐색(BFS/DFS); Game of 24 압승, 비용 ~100×
  • MemGPT: OS 비유(RAM=in-context↔disk=external); 70%→경고만, 100%→50% evict+recursive summary; 관리는 LLM function call
  • A-Mem(NeurIPS 2025): note \(m_i=\{c,t,K,G,X,e,L\}\); cosine top-k는 후보만, 링크는 LLM이 생성; evolution은 이웃만 갱신
  • Multi-agent: persona만 달라도 다른 agent; MDAgents=complexity check→solo/group/teams; orchestration=강한 lead(Opus)+싼 sub(Haiku); routing=비용 최적화
LLaVA projector: \( \mathbf{Z}_v\in\mathbb{R}^{L\times D'} \xrightarrow{\mathbf{W}} \mathbf{H}_v\in\mathbb{R}^{L\times D} \) (vision token은 벡터, vocabulary 없음)
"LLaVA stage 2 fine-tunes the vision encoder." → False (always frozen)
"Reflexion updates the LLM's weights from feedback." → False (verbal memory only)
"MemGPT evicts at 70% usage." → False (70%=warning, 100%=evict 50%+summary)
ReAct=interleave 프롬프팅 / Reflexion=언어 피드백 메모리 / PS=zero-shot 계획 / ToT=트리 탐색(+고비용)
#12 RL with LLMs 수식 최다
  • 매핑: policy=LLM / state=input tokens / action=next-token generation; 목표=return(누적 보상) 최대화
  • \(V^\pi\)=state 조건부, \(Q^\pi\)=state+action 조건부, \( V^\pi(s)=\mathbb{E}_{a\sim\pi}[Q^\pi(s,a)] \); γ는 infinite-horizon 발산 방지용
  • PG 유도 3요소: log-derivative trick / 환경항(\(\rho_0\), transition) gradient=0 / Monte Carlo 표본평균(on-policy)
  • 분산 감소 3종: reward-to-go(t 이후만) / baseline \(b(s_t)\)(state만 의존, 기댓값 불변·분산↓) / advantage \(A=Q-V\)
  • PPO-Clip: ratio를 \([1\!-\!\epsilon,1\!+\!\epsilon]\)로 clip; A>0→상한(ceiling), A<0→하한; value model 필요; 멀어짐 완전 보장은 안 됨 (수업 표기 "Proximal Point Optimization"도 동일물)
  • TRL: rollout은 torch.no_grad(샘플링만); 별도 value 모델 대신 value head(LM head=vocab 분포, value head=스칼라)
  • GRPO: value model 제거 — 그룹 G개 rollout의 mean/std로 advantage; clip은 PPO 그대로+KL(ref, 다른 estimator); 동기=value 의존 제거(stability 아님); 생성 비용↑
  • 변형: Dr.GRPO=std 제거 / REINFORCE++=query-wise; GLM-5: SFT→Reasoning RL→Agentic RL→General RL(alignment 마지막), 여전히 GRPO; On-policy distillation: student가 생성한 데이터에 teacher가 supervision→forgetting 강건
\( \nabla J=\mathbb{E}[\sum_t\nabla\log\pi_\theta(a_t|s_t)A^\pi] \) · \( \hat{A}_{i}=\frac{r_i-\text{mean}(\{r\})}{\text{std}(\{r\})} \)
"A baseline may depend on the action." → False (state-only, else biased)
"When A>0 the clip term is a lower bound." → False (upper bound; lower는 A<0)
"GRPO's motivation is preventing abrupt updates like PPO." → False (value 제거가 동기)
REINFORCE=R(τ) 그대로 / PPO=clip+value model / GRPO=group 정규화+value 없음(clip·KL은 유지)
#13 Jailbreak Special Lecture
  • Training-level 방어: Red Teaming(전문가→학습 데이터, 비용↑) / RBRM=rule-based reward로 RLHF 보강(대체 아님, 선호 학습 아님)
  • DAN=수작업 역할부여(refusal suppression류); good bot=refuse, bad bot=on-topic 응답
  • Failure mode 2(Jailbroken, NeurIPS 2023 Oral): ① Competing objectives — prefix injection(next-token vs safety)·refusal suppression(instruction following vs safety) ② Mismatched generalization — Base64; Comb_3가 SOTA ASR
  • GCG(Zou 2023): gradient 기반 suffix — 의미 없음 → perplexity filter에 쉽게 탐지
  • AutoDAN(ICLR 2024): hierarchical genetic algorithm; 초기화=handcrafted DAN(중요!), fitness=proxy target likelihood, crossover=문장 교환/word scoring mutation; fluent→perplexity 방어 생존, 전이성↑; ASR=keyword 기반(GPT 판정은 Recheck)
  • COLD-Attack(ICML 2024): COLD와 알고리즘 exactly the same(energy+Langevin on logits), 제약만 attack/fluency/similarity/lexical로 교체; GCG·AutoDAN보다 빠름
  • SafeDecoding(ACL 2024): LoRA expert와 contrastive decoding으로 refusal 토큰↑; first m tokens만; benign에선 보정항 cancel; inference-level
  • Gradient Cuff(NeurIPS 2024): refusal loss \(f<0.5\) + zeroth-order gradient norm>t 2단계; benign=flat / malicious=steep; SafeDecoding과 trade-off(우열 없음)
SafeDecoding: \( P_n(x)=p_\theta+\alpha(p_{\theta'}-p_\theta) \) · refusal loss: \( \phi_\theta(x)=1-\mathbb{E}_{y}[JB(y)] \), \(JB(y)\!=\!1\)=refusal
"Base64 attack exploits competing objectives." → False (mismatched generalization)
"GCG evades perplexity-based detectors." → False (정반대 — 쉽게 탐지됨)
"\(JB(y)=1\) when jailbreak succeeds." → False (1=refusal; "Sure, here is..."=0)
GCG=gradient·탐지됨 / AutoDAN=GA·fluent / COLD-Attack=Langevin·최고속 · 방어: SafeDecoding(디코딩 보정) vs Gradient Cuff(탐지·거부)
#14 AI Text Detection Special Lecture
  • 3갈래: ① Zero-shot(DetectGPT/Fast-DetectGPT) ② Training-based(RoBERTa, ReMoDetect) ③ Watermarking(생성 시점에 패턴 삽입)
  • DetectGPT(ICML 2023): LLM 텍스트는 \(\log p_\theta\)의 local maxima(negative curvature); perturbation은 별도 mask-fill LM(T5), 의미 보존; black-box 아님(\(\log p_\theta\) 필요); ~100 perturbations에서 포화; OOD에서 supervised보다 우수(in-dist는 on-par); AUROC=전 threshold 적분
  • Curvature 해석: Taylor + Hutchinson(조건: \(\mathbb{E}[z]=0,\text{Var}=1\); 대칭성은 추가 가정) → score \(=-\frac12\text{tr}(H_f)\), full Hessian 불필요
  • Fast-DetectGPT(ICLR 2024): \(p_\theta(\tilde x)\to p_\theta(\tilde x|x)\), conditional independent sampling(원본 prefix 조건)으로 N×L→L inferences; ×340(22h→4min), 성능 유지~상회
  • ReMoDetect(NeurIPS 2024): reward model은 LLM 생성문(LGT)에 biased→탐지기로; continual preference tuning은 LGT를 preferred로(직관 반대!)+replay buffer; Mixed responses(LLM≻Mixed≻Human); DPO 모델에도 적용 가능
  • Watermark(Kirchenbauer, ICML 2023): 이전 토큰 hash→green/red 매 위치 재분할; hard=red 금지 / soft=green logit+δ; 사람에겐 imperceptible; 탐지에 모델 접근 불필요(z-test만); τ=4→FP≈3×10⁻⁵, 16+ 토큰; SynthID 채택
\( \mathbf{d}=\log p_\theta(x)-\mathbb{E}_{\tilde x}[\log p_\theta(\tilde x)] \) · \( z=\frac{2(|s|_G-T/2)}{\sqrt{T}} \)
"DetectGPT works fully black-box." → False (needs \(\log p_\theta\); watermark 탐지는 모델 불필요)
"Fast-DetectGPT is faster because it uses fewer samples." → False (conditional independent sampling 덕분)
"In continual preference tuning, human text is preferred." → False (LGT가 preferred)
Zero-shot=곡률(모델 logprob 필요) / Trained=reward model 활용 / Watermark=생성에 개입·탐지는 통계 검정만

🔀 교차 비교 — 챕터를 넘나드는 헷갈림 정리

COLD(#7) vs COLD-Attack(#13) — 알고리즘은 완전히 동일(에너지 함수 + 로짓 공간 Langevin dynamics + discretization). COLD-Attack은 제약(에너지 항)만 attack·fluency·similarity·lexical로 바꿔 jailbreak에 적용. "COLD-Attack proposes a new sampling algorithm" → False.
Langevin dynamics 등장 위치 — #7 COLD와 #13 COLD-Attack 둘뿐. 원 논문은 Welling (ICML 2011). \( \tilde{\boldsymbol{y}}^{(n+1)}=\tilde{\boldsymbol{y}}^{(n)}-\eta\nabla E+\epsilon \), 노이즈 \(\epsilon\sim\mathcal{N}(0,\sigma)\)이 있어 단순 gradient descent가 아니라 샘플링.
학습 방식 3종: Self-RAG vs Search-R1 vs ToolLLM — Self-RAG=SFT(GPT-4 주석→critic→generator, reflection 토큰) / Search-R1=RL(outcome reward만, 주석 불필요, interleaved 검색 창발) / ToolLLM=SFT(instruction tuning)(ChatGPT가 DFSDT로 푼 golden path). 공통점: Self-RAG·Search-R1 모두 검색된 토큰은 loss에서 제외(masking).
DetectGPT curvature(#14) vs COLD energy(#7) — 둘 다 "함수의 모양"을 보지만 정반대 용도: DetectGPT는 \(\log p_\theta\)의 local maxima 여부(음의 곡률)를 측정해 탐지, COLD는 에너지 \(E\)를 내려가며(gradient) 텍스트를 생성. 측정 vs 최적화.
RLHF · RLAIF · Constitutional AI 관계 — RLHF=사람 선호 레이블(응답은 AI 생성, 레이블만 사람). RLAIF=선호 레이블도 AI. Constitutional AI = SL-CAI(critique→revision SFT) + RL-CAI(=RLAIF의 구체화, principle-conditioned RM). 사람의 역할은 헌법 작성뿐. RL-CAI의 시작점은 helpfulness-only RLHF 모델(베이스 LM 아님).
PPO가 등장하는 모든 맥락 — ① RLHF/InstructGPT(#8·#12, Ouyang NeurIPS 2022)의 학습 알고리즘 ② TRL 구현(#12, value head·no_grad rollout) ③ Search-R1(#9)의 목적함수(PPO/GRPO 호환, KL 포함) ④ GRPO(#12)가 clip 구조를 상속 ⑤ GLM-5(#12)도 결국 GRPO 계열. "GRPO is not policy gradient" → False (둘 다 policy optimization).
Reward model의 4가지 얼굴 — ① Test-time compute의 Best-of-N(#7) ② RLHF 선호 학습(#8·#12) ③ RBRM(#13)은 예외: rule-based, 선호 학습 아님 ④ ReMoDetect(#14)는 reward model의 LGT bias를 탐지기로 역이용.
"Contrastive"의 두 의미 — #9 DPR/Contriever의 contrastive loss(positive 가깝게, negative 멀게)와 #13 SafeDecoding의 contrastive decoding(expert−base 확률 차로 보정)은 전혀 다른 개념.
Training-free 계열 총정리 — BM25(#9), COLD(#7), prompting 기반 tool use(#10), ReAct/ToT(#11, 프롬프팅), DetectGPT·Fast-DetectGPT(#14), inference-based RAG(REPLUG의 LM·IRCoT). 반대편(학습): DPR·Contriever·Self-RAG·Search-R1·Toolformer·ToolLLM·ReMoDetect·SafeDecoding expert.
자동 공격 3종 최적화 방식 — GCG=greedy gradient search(이산) / AutoDAN=genetic algorithm(DAN 초기화) / COLD-Attack=Langevin(연속 로짓). 유창성: GCG ✗(탐지됨) / AutoDAN ✓ / COLD-Attack ✓(+빠름).

∑ 수식 한 곳 모음 (이름 → 수식 → 한 줄 의미)

AR factorization (#7)
$$ p(\boldsymbol{y}\mid\boldsymbol{x})=\prod_{t=1}^{T}p(y_t\mid\boldsymbol{x},y_{한 토큰씩, 모든 이전 토큰 조건부 — 그래서 inference는 sequential.
Temperature (#7)
$$ p(y_t=v)=\frac{\exp(z_v/\tau)}{\sum_{v'}\exp(z_{v'}/\tau)} $$
softmax 이전 로짓 rescale; τ↓ peaky(deterministic), τ↑ smooth.
COLD energy (#7/#13)
$$ E(\boldsymbol{y})=-\sum_i\lambda_i f_i(\boldsymbol{y}),\qquad p(\boldsymbol{y})=\frac{e^{-E(\boldsymbol{y})}}{Z} $$
제약 만족↑ → 에너지↓ → 확률↑ (Boltzmann form).
Langevin dynamics (#7/#13)
$$ \tilde{\boldsymbol{y}}^{(n+1)}=\tilde{\boldsymbol{y}}^{(n)}-\eta\,\nabla_{\tilde{\boldsymbol{y}}}E(\tilde{\boldsymbol{y}}^{(n)})+\epsilon^{(n)},\quad \epsilon^{(n)}\sim\mathcal{N}(0,\sigma) $$
gradient step + 가우시안 노이즈 → n→∞에서 목표 분포 샘플링.
PageRank (#9)
$$ PR(A)=\frac{1-d}{N}+d\sum_{B_i\in\text{In}(A)}\frac{PR(B_i)}{L(B_i)} $$
중요도=이웃 중요도/링크수 합; \(1-d\)=random jump 확률; power iteration 수렴.
BM25 (#9)
$$ \text{score}(D,Q)=\sum_i \text{IDF}(q_i)\cdot\frac{f(q_i,D)(k_1+1)}{f(q_i,D)+k_1\big(1-b+b\frac{|D|}{\text{avgdl}}\big)},\;\; \text{IDF}(q_i)=\ln\!\Big(\frac{N-n(q_i)+0.5}{n(q_i)+0.5}+1\Big) $$
IDF=corpus 전체 희소성 × TF=대상 문서 빈도(길이 보정); 무학습.
DPR loss (#9)
$$ \mathcal{L}=-\log\frac{e^{\text{sim}(q,p^+)}}{e^{\text{sim}(q,p^+)}+\sum_j e^{\text{sim}(q,p_j^-)}},\qquad \text{sim}(q,p)=E_Q(q)^{\top}E_P(p) $$
contrastive: positive 유사도↑, negative↓; Contriever도 같은 loss(데이터만 자기지도).
REPLUG ensemble (#9)
$$ p(y\mid x,\mathcal{D}')=\sum_{d\in\mathcal{D}'}p(y\mid d\circ x)\cdot\lambda(d,x),\qquad \lambda(d,x)=\frac{e^{s(d,x)}}{\sum_{d'}e^{s(d',x)}} $$
문서별 독립 prepend 출력을 retrieval score로 가중 평균(블랙박스 LM).
REPLUG LSR (#9)
$$ \mathcal{L}_{LSR}=\mathbb{E}_x\,\mathrm{KL}\big(P_R(d\mid x)\,\|\,Q_{LM}(d\mid x,y)\big) $$
retrieval likelihood ↔ LM likelihood의 KL 최소화 — LM은 frozen, 검색기만 학습.
Self-RAG generator loss (#9)
$$ \max_{\mathcal{M}}\;\mathbb{E}_{(x,y,r)\sim\mathcal{D}_{gen}}\big[\log p_{\mathcal{M}}(y,r\mid x)\big] $$
reflection 토큰 \(r\) 포함 표준 likelihood; 검색 청크는 loss에서 마스킹.
Search-R1 objective (#9)
$$ \max_{\pi_\theta}\;\mathbb{E}_{x,\,y\sim\pi_\theta(\cdot|x;R)}\big[r_\phi(x,y)\big]-\beta\,\mathbb{D}_{KL}\big[\pi_\theta(y|x;R)\,\|\,\pi_{ref}(y|x;R)\big] $$
RLHF 목적과 동형(+검색기 R 개입); outcome reward만, retrieved token masking.
Toolformer loss (#10)
$$ L_i(\mathbf{z})=-\sum_{j=i}^{n}w_{j-i}\log p_M(x_j\mid\mathbf{z},x_{1:j-1}) $$
호출 위치 \(i\) 이후의 가중 next-token loss; \(\mathbf{z}\)=프리픽스로 넣는 API 텍스트.
Toolformer filtering (#10)
$$ L_i^+=L_i(e(c_i,r_i)),\quad L_i^-=\min\big(L_i(\varepsilon),L_i(e(c_i,\varepsilon))\big),\quad L_i^--L_i^+\ge\tau\Rightarrow\text{keep} $$
호출+결과가 loss를 τ 이상 낮춰야 keep; τ 클수록 적게 살아남음.
Policy gradient (#12)
$$ \nabla_\theta J(\pi_\theta)=\mathbb{E}_{\tau\sim\pi_\theta}\Big[\sum_{t=0}^{T}\nabla_\theta\log\pi_\theta(a_t\mid s_t)\,R(\tau)\Big] $$
log-derivative trick + 환경항 gradient=0; MC 표본 평균으로 추정(on-policy).
Baseline / Advantage (#12)
$$ \mathbb{E}_{a_t\sim\pi_\theta}\big[\nabla_\theta\log\pi_\theta(a_t|s_t)\,b(s_t)\big]=0,\qquad A^\pi(s,a)=Q^\pi(s,a)-V^\pi(s) $$
state-only baseline은 기댓값 불변·분산↓; advantage=평균 대비 우수성.
PPO-Clip (#12)
$$ L=\min\!\Big(\frac{\pi_\theta(a|s)}{\pi_{\theta_k}(a|s)}A^{\pi_{\theta_k}},\;\mathrm{clip}\Big(\frac{\pi_\theta(a|s)}{\pi_{\theta_k}(a|s)},1\!-\!\epsilon,1\!+\!\epsilon\Big)A^{\pi_{\theta_k}}\Big) $$
A>0→min이 상한, A<0→하한; 급격한 정책 변화 억제(완전 보장은 아님).
GRPO advantage & KL (#12)
$$ \hat{A}_{i,t}=\frac{r_i-\mathrm{mean}(\{r_1..r_G\})}{\mathrm{std}(\{r_1..r_G\})},\qquad \mathbb{D}_{KL}=\frac{\pi_{ref}}{\pi_\theta}-\log\frac{\pi_{ref}}{\pi_\theta}-1 $$
그룹 내 정규화로 value model 대체; KL은 별도 estimator(항상 ≥0).
AutoDAN objective (#13)
$$ \max_{J_i}\;\prod_{t=1}^{k}p\big(r_{m+t}\mid J_i,Q_i,r_{m+1:m+t-1}\big) $$
proxy target("Sure, here is...")의 likelihood를 GA의 fitness로 최대화.
SafeDecoding (#13)
$$ P_n(x)=p_\theta(x\mid x_{1:n-1})+\alpha\big(p_{\theta'}(x\mid x_{1:n-1})-p_\theta(x\mid x_{1:n-1})\big) $$
expert(LoRA)와의 차이로 refusal 확률 증폭; first m tokens만; benign에선 cancel.
Refusal loss / Gradient Cuff (#13)
$$ \phi_\theta(x)=1-p_\theta(x),\quad p_\theta(x)=\mathbb{E}_{y\sim T_\theta(x)}[JB(y)],\qquad g_\theta(x)=\tfrac{1}{P}\sum_i\tfrac{f(e(x)\oplus\mu\mathbf{u}_i)-f(e(x))}{\mu}\mathbf{u}_i $$
\(JB=1\)이면 refusal; zeroth-order로 기울기 norm 추정 — malicious=steep.
DetectGPT discrepancy (#14)
$$ \mathbf{d}(x,p_\theta,q)=\log p_\theta(x)-\mathbb{E}_{\tilde{x}\sim q(\cdot|x)}[\log p_\theta(\tilde{x})] \;=\; -\tfrac12\,\mathrm{tr}\big(H_f(x)\big) $$
크면 machine(local maximum); Taylor+Hutchinson으로 Hessian trace와 연결.
Fast-DetectGPT (#14)
$$ \mathbf{d}(x,p_\theta,q_\varphi)=\frac{\log p_\theta(x|x)-\tilde{\mu}}{\tilde{\sigma}},\qquad \tilde{\mu}=\mathbb{E}_{\tilde{x}\sim q_\varphi(\cdot|x)}[\log p_\theta(\tilde{x}|x)] $$
조건부 독립 샘플링으로 N×L→L inferences (×340 가속).
Watermark z-test & soft (#14)
$$ z=\frac{2(|s|_G-T/2)}{\sqrt{T}},\qquad p_k^{(t)}=\frac{\exp(l_k^{(t)}+\delta\,\mathbb{1}[k\in G])}{\sum_i\exp(l_i^{(t)}+\delta\,\mathbb{1}[i\in G])} $$
green 비율의 one-proportion z-test(τ=4→FP≈3×10⁻⁵); soft=green logit에 δ만 가산.

✅ 마지막 점검 — 이거 헷갈리면 그 챕터 다시 봐

  1. Temperature는 softmax 이전 로짓에 적용, τ↓=deterministic. Top-p 후보 개수는 동적. (#7)
  2. Toolformer는 \(L_i^- - L_i^+ \ge \tau\)일 때 keep (반대 방향이면 False); 사실성 검사 안 함. (#10)
  3. \(A=Q-V\) (V−Q 아님); baseline은 state에만 의존해야 unbiased. (#12)
  4. PPO clip: A>0 → min이 상한(ceiling), A<0 → 하한. GRPO의 동기는 stability가 아니라 value model 제거. (#12)
  5. Self-RAG=SFT / Search-R1=RL; 둘 다 검색된 토큰은 loss에서 마스킹. Search-R1 원버전 보상은 outcome만. (#9)
  6. RetRobust NLI: passage=premise, Q+A=hypothesis; not-entailed면 재검색이 아니라 direct answering. (#9)
  7. Good bot=refuse; prefix injection=next-token vs safety / refusal suppression=instruction following vs safety / Base64=mismatched generalization. (#13)
  8. DetectGPT는 black-box 아님(\(\log p_\theta\) 필요); watermark 탐지는 모델 접근 불필요(z-test만). perturbation은 별도 T5. (#14)
  9. RBRM=rule-based(선호 데이터 학습 ✗); LC win rate=회귀로 length 제거; position bias swap은 2× cost, 불일치=Tie. (#8/#13)
  10. MemGPT: 70%=경고만, 100%=50% evict+recursive summary; LLaVA encoder는 두 단계 모두 frozen; Reflexion은 weight update 없음. (#11)
