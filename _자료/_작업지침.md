## 본문 작성 원칙
1. **슬라이드가 뼈대, 전사가 살**: 슬라이드 순서·구조를 따라가되, 교수가 강의에서 덧붙인 직관·예시·강조("이건 시험에..." 류 포함)를 `.prof-quote`로 살린다.
2. **빠짐없이**: 배정된 슬라이드 범위의 모든 개념·논문·수식·실험 결과를 다룬다. 논문은 (저자/학회/연도) 표기.
3. 한국어 설명 + **시험에 나올 영어 용어 그대로 병기**. 함정 포인트와 예상문제는 영어 서술 위주(시험이 영어이므로).
4. **수식은 KaTeX**: `$$...$$`(블록), `\(...\)`(인라인). 슬라이드의 수식은 전부 재현하고 기호 의미를 풀이.
5. 비교 가능한 것은 **반드시 비교표**로 (예: PPO vs GRPO, Toolformer vs ToolLLM, ReAct vs Reflexion vs ToT, BM25 vs DPR vs Contriever).
6. 모든 주요 섹션에 `.box-warn` **함정 포인트**: 기출 해설 패턴("Incorrect. ...")처럼 흔한 오해를 T/F 형태로. "X라고 하면 False, 왜냐하면..."
7. `.box-key` 시험 핵심: 그 섹션에서 출제된다면 무엇이 나올지.

## 문제 섹션 (각 챕터 필수)
- **기출 배정**: 2025 중간 기출 중 해당 챕터 관련 문항은 **원문 그대로**(영어) + 정답·해설 포함, `src-real` 뱃지.
  - #7 Decoding ← 기출 Problem 5 전체(Q1~Q5: autoregressive/temperature/top-p/speculative/test-time compute) + Problem 1(x) DBS
  - #8 AI Feedback ← 기출 Problem 4-5(evaluation bias, LC win rate)
  - #9~#14 ← 직접 기출 없음(중간고사 이후 범위) → 예상문제만
- **예상문제**: 챕터당 **T/F 6~10개 + Pick-all 4~8개**, 기출과 똑같은 영어 문체·형식·난이도. 해설도 기출 패턴("Correct./Incorrect. + 이유").
- 정답은 `<details>` 토글로 가림.
